## Things that surprised me (read these)

1. **FP-03 submission-flow anomaly (the big one).** The script assumed a signed-out
   teacher can fill in a lesson URL, hit Submit, then be gated to sign-in and have it
   auto-fire. Reality: clicking "Add a new lesson" gates to sign-in **immediately**, on
   both live and #585's preview. A background read-only investigation
   (`scratchpad/fp03-submission-gate-investigation.md`) found why: the fill-first flow was
   **removed in commit `f218800`** (Phase 8b intent-first chooser). Today's chooser gates
   via a handler guard (`SubmissionPage.tsx:27-34`; `/submit*` routes are public, not
   ProtectedRoute). #585's actual change is confined to `AuthModal.tsx` (+2 tests) and
   makes the pending-submit ref survive sign-in — but that path is **only reachable via a
   direct `/submit/new` deep-link / bookmark / refresh while signed out**, never via the
   primary button flow. #585 is a small, safe, test-covered fix (merged), BUT there's an
   **open product question for Fable: is FP-03 worth keeping / should the primary flow let
   teachers fill first?** ("sign in and we'll submit your lesson" is the PR author's
   paraphrase, not literal UI text.)

2. **Script number correction: Indoor badge = 607, not 417.** The live search already
   folds "Both" into Indoor/Outdoor (`_match_location`, `20260620000000_search_lessons_w1b.sql:137`).
   417 is the literal-Indoor bucket; indoor-friendly (Indoor∪Both) = 607 on PROD. FP-18
   (collapse Location to two checkboxes) is therefore **purely cosmetic** — 0 active lessons
   have a blank location, so both-checked == neither-checked == all 703.

3. **#591 needed a hand-resolved merge conflict.** Only `src/pages/UserProfile.tsx` (both
   #586 and #591 edited the `useEnhancedAuth()` destructure — additive, combined both). A
   *semantic* follow-on: #591 widened `AuthContextValue` with `profileError`/`retryAuth`,
   which made #586's `UserProfile.test.tsx` mock incomplete (git auto-merge couldn't see
   it). Fixed the mock; local type-check + all 7 UserProfile tests pass; CI re-validated.
