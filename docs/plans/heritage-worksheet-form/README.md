# Heritage Worksheet — Curriculum Team Review

This folder holds an interactive tool for filling out the Stage 1 Heritage
Worksheet. The whole tool is a single HTML file — everything it needs
(layout, code, worksheet text) is already inside it.

## How to open it

Double-click **`heritage-worksheet-tool.html`**. It opens in your default web
browser. Chrome, Safari, Firefox, and Edge all work; no installation, no
internet connection required.

If your browser doesn't open it automatically, right-click the file and
choose "Open With" → your browser of choice.

## What it does

For each of the 88 heritage values, the tool asks you to make one decision
(keep, merge, split, drop, or add as new) and shows a plain-English
recommendation plus any alternatives. Most entries are "just confirm the
proposal" — one click. A handful require your team's judgment.

There are also five framing decisions (Decisions 1–5 in the panel at the
top) and five cluster decision summary textareas (one per regional
cluster) for capturing the reasoning behind the cluster's shape.

## How your progress is saved

Your answers save automatically as you work, inside your browser, on this
computer. The "Saved locally" badge in the bottom-right corner is there to
reassure you. You can close the tab and come back later — your work will
still be there when you reopen the file.

A few important caveats about local saving:

- Your progress is tied to **this browser on this computer**. Switching to
  another browser or another machine won't bring your work with you
  automatically.
- If you clear browser data ("Clear browsing history" with site data
  included), the tool will forget your answers.
- To move your work to another machine (or to back it up), use the
  **Save & Export ↓** button — it produces a markdown file you can save
  anywhere. Then use **Import…** under the ⚙ Advanced menu on the other
  machine to load it back.

## When you're done — exporting your work

Click **Save & Export ↓** in the top bar. A markdown file called
`2026-05-10-metadata-rebuild-stage1-heritage-worksheet.md` will download
to your usual Downloads folder. Send that file back to the project
maintainer.

The exported markdown is the full original worksheet with all your
decisions filled in. The project's parser reads it directly — no
conversion needed.

## Taking a break and resuming later

Just close the tab. When you reopen the HTML file, your progress will
appear automatically. If you want to keep multiple draft versions or
move work between machines, use Save & Export to download a checkpoint,
then Import (under ⚙) to load any saved checkpoint back.

## What if I get stuck on an entry?

Two things help:

1. **Read the "Why we recommended this" notes.** Every entry has a
   "Full reviewer notes" expander with detailed reasoning. The notes
   explain how many lessons use the value, what patterns appear in lesson
   bodies, and which audit signals were flagged.
2. **Skip and come back.** You can leave an entry's verdict empty and
   move on. The progress bar tracks how many entries are still pending.
   Use the **Needs my call** filter to find them later.

If a §9.1 framing decision is what's blocking you, fill it in the
Decisions panel at the top first — those decisions cascade into specific
§9.1 entries below.

## Questions, pushback, "I disagree with the recommendation"

The pre-handoff recommendations are starting points, not finished
decisions. Pushback is the whole point. Use the **Reviewer note** input
on each entry to flag concerns inline (they'll appear as comments in the
exported markdown), or reach out to the project maintainer directly.

## Browser quirks (rare)

- The tool stores up to a few hundred KB of state in your browser. Some
  browsers in "private" / "incognito" mode wipe this when the window
  closes — don't use private mode for sustained work, or do, but export
  often.
- The Import button uses a standard file picker; if your browser blocks
  file uploads (some corporate policies do), open the file in a personal
  browser profile instead.
